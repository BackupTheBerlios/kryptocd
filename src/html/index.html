Secure Backups

KryptoCDs purpose is to provide an easy way of making encrypted backups. It is a frontend for some standard GNU tools. KryptoCD itself doesn't contain any cryptographic software, you need to install GnuPG, which is a trustworthy encryption software.

Why should I encrypt my backup?

I presume, that you are using Unix. This is a quite secure operating system - other users can't read your files i.e. But if you make a backup for example on CD, everbody can take the CD, mount it and read your files. This breaks the security concept completely, so you should better encrypt your backups.

How does KryptoCD work?

KryptoCD works not at all - it's in the planning phase. It will use GnuPG to encrypt your backup. You will choose the files or directories to backup, and KryptoCD will then compress (tar, bzip2) and encrypt (gnupg) them. Afterwards will this encrypted backup be burned on CD (cdrecord).

Where can I get KryptoCD

KryptoCD is Open Source. You will get it for free at BerliOS, when it is released. You will need KDE 2 to run KryptoCD.
