Sichere Backups

Der Zweck von KryptoCD ist, auf einfache Art und Weise verschlüsselte Sicherheitskopien (Backups) von wichtigen Dateien zu erstellen. Es ist ein Frontend für einige gebräuchliche GNU Tools und enthält selbst keine kryptografische Software. Sie benötigen für die Verschlüsselung GnuPG. GnuPG ist vertrauenswürdig, seine Entwicklung wird vom Bundesministerium für Wirtschaft und Technologie (BMWi) gefördert.

Warum sollte man Sicherungskopien verschlüsseln?

Vermutlich benutzen Sie mit Unix ein recht sicheres Betriebssystem. Andere Benutzer können beispielsweise Ihre privaten Daten nicht lesen. Aber wenn sie eine Sicherungskopie dieser Daten zum Beispiel auf eine CD brennen, kann jeder diese CD in sein Dateisystem einbinden und Ihre Daten lesen.

Durch unverschlüsselte Sicherungskopien entsteht also eine Sicherheitslücke. Sie können zur Lösung dieses Problems natürlich Ihre Sicherheitskopien an einem sicheren Ort wie einem Tresor aufbewahren, aber Sie können Ihre Daten auch einfach verschlüsseln.

Sie können KryptoCD natürlich nicht nur für Sicherheitskopien verwenden: Wenn Sie jemandem per CD wichtige Daten zukommen lassen wollen, können Sie diese dafür auch verschlüsseln. Der Empfänger muss allerdings ebenfalls GnuPG und andere GNU Tools benutzen, um die Daten wieder entschlüsseln zu können.

Wie funktioniert KryptoCD?

KryptoCD funktioniert noch garnicht, es befindet sich nämlich noch in der Planungsphase. Es wird GnuPG zu Verschlüsselung Ihrer Sicherungskopien verwenden. Sie können dann Dateien oder ganze Verzeichnisse auswählen, diese werden komprimiert (tar, bzip2), verschlüsselt (gnupg) und anschließend auf CDs gebrannt (cdrecord).

Wo ist KryptoCD erhältlich?

KryptoCD ist Open Source. Sie können KryptoCD, nachdem es erschienen ist, bei BerliOS herunterladen. Sie werden KDE 2 benötigen, um KryptoCD benutzen zu können.
